# 背景
```note
如果在对数的运算过程中，数的值过于大，超出了long long int，那么我们可能会考虑高精度计算，可是一旦才用高精度，加法和减法的时间复杂度就会变成O(k)，k是数字的位数（因为高精度相当于把每一位单独算一次，有多少位就要算多少次）,而乘法就会是O(k^2);一般而言，对于这种特别大的数字，我们只需判断这个数对一个比较大数取余的结果即可，而对于这种情况就可以用到同余原理

```
# 同余原理
```
1.(a+b)%m = ((a%m) + (b%m))%m
2.(a*b)%m = ((a%m)*(b%m))%m
3.(a-b)%m = (a%m-b%m+m)%m //加m是为了保证余数的结果非负
```
